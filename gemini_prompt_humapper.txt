Evidence basis (must follow):
- SNOMED CT semantic tags: "disorder" is an abnormal clinical state; "finding" is an observation/assessment;
  "situation with explicit context" carries context (history/family/risk/planned/etc.);
  "morphologic abnormality" denotes pathological morphology context.
- HL7 FHIR Condition status logic:
  clinical_status(active/recurrence/relapse/inactive/remission/resolved),
  verification_status(unconfirmed/provisional/differential/confirmed/refuted/entered-in-error).

Core rules (high priority):
1) Semantic-tag triage (when phrase ends with "(...)"):
   - "(disorder)" -> severity="sick" by default (if not negated).
   - "(morphologic abnormality)" or "(morphological abnormality)" -> severity="sick" (if not negated).
   - "(finding)" -> severity="warning" by default.
   - "(situation)" -> severity="warning" by default (context/risk/history/planned); do not auto-upgrade to sick without acute evidence.
   - "(procedure)" -> usually not a diagnosis; use "warning" only when complication/adverse pathology is explicit, otherwise skip.
   - "(person)" -> not a diagnosis; skip unless explicitly used as pathology context (rare).

2) FHIR status override (stronger than default tag):
   - clinical_status: active|recurrence|relapse -> sick
   - clinical_status: inactive|remission -> warning
   - clinical_status: resolved -> healthy (unless persistent active complication is explicitly present)
   - verification_status: confirmed -> keep current severity
   - verification_status: provisional|differential|unconfirmed|suspected|possible -> cap at warning (unless emergency red flags)
   - verification_status: refuted|entered-in-error -> healthy or skip

3) Negation/absence always de-escalates:
   if statement is clearly absent/negative/ruled-out/no-evidence, do not output sick.

4) Emergency/pathology escalation to sick:
   infarction, stroke, sepsis, shock, acute respiratory failure, acute renal/hepatic failure,
   pulmonary embolism, active major bleeding, perforation, acute fracture/dislocation,
   confirmed malignant neoplasm/cancer/tumor.

5) Organ mapping safety:
   - Use only allowed organ ids.
   - Map organ only if anatomically/clinically justified by the same statement.
   - NEVER assign a default organ (especially never default to heart).
   - If organ cannot be mapped reliably, skip that item instead of inventing organ ids.
   - Dental/oral terms (gingivitis, periodontitis, dental caries, tooth, gum, oral cavity, palate, buccal mucosa, dental care)
     map to oral_cavity unless the statement explicitly describes TMJ or salivary glands.
   - Sinusitis / rhinosinusitis / rhinitis / nasal congestion map to nasal_sinuses, not to heart and not to whole skull.
   - When sinus or nasal disease is stated, prefer the smallest plausible focus (nose + paranasal sinuses) and avoid generic skull highlighting.
   - TMJ / temporomandibular statements map to tmj, not heart and not generic skull.
   - Salivary/parotid/submandibular/sublingual statements map to salivary_glands.
   - Do not map oral/dental disease to skull just because the anatomy is in the head region.
   - Do not map to heart unless there is explicit cardiac evidence (heart/cardiac/myocardial/coronary/arrhythmia/atrial/ventricular/pericardial terms).
   - General cranial/head/facial words alone are not enough to infer heart or skull pathology.

6) Non-clinical social/context statements:
   education level, transport access, housing, occupation/service, administrative context, etc.
   are not direct organ pathology. Classify as warning only if clinically relevant risk is explicit; otherwise skip.

7) Deduplication/confidence:
   - No duplicates.
   - Confidence range 0.55..0.99.
   - Higher confidence for explicit semantic tag + explicit organ + explicit status markers.

Language handling:
- Input can be English or Russian.
- Keep disease names as they appear in source text when possible.
- Be robust to bilingual medical UI text where the visible diagnosis may be Russian but the source note may still contain English terms.
- When English and Russian terms point to the same concept, keep one clinically coherent result only.
